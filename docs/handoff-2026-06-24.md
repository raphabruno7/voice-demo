## Arquitectura do sistema

```
Vercel Cron 07:00 UTC → /api/cron/health-check
  ├── runAllChecks() [Promise.allSettled — 10 serviços em paralelo]
  │   ├── Hume: GET /v0/evi/configs/{id} — autenticação OAuth
  │   ├── LiveKit: RoomServiceClient.listRooms()
  │   ├── ElevenLabs: GET /v1/convai/agents/{id}
  │   ├── Vapi: GET /assistant/{id} (ok em 401/403, fail só em 5xx)
  │   ├── Retell: GET /v2/get-agent/{id}
  │   ├── Twilio: GET /2010-04-01/Accounts/{sid}.json
  │   ├── Google Calendar: listUpcomingEvents() [janela 60s]
  │   ├── Supabase: SELECT id FROM health_checks LIMIT 1
  │   ├── Railway: GET $LIVEKIT_AGENT_HEALTH_URL/health [porta 8081]
  │   └── Fly.io: GET $TWILIO_AGENT_HEALTH_URL/health [porta 8080]
  ├── INSERT resultados em Supabase health_checks
  ├── DELETE registos > 30 dias
  ├── SE fail → sendHealthEmail(results, 'alert')
  └── SEMPRE → sendHealthEmail(results, 'daily')

/status [Server Component, protegido por middleware]
  ├── getLatest() — último resultado por serviço (100 rows, dedup JS)
  └── getHistory() — 30 dias, barras CSS inline
```

**Thresholds:** ok < 2 000ms | degraded 2 000–5 000ms | fail > 5 000ms ou erro

### 2. Aplicar migração Supabase
Abrir Supabase dashboard → SQL Editor → executar:
```sql
-- conteúdo de supabase/migrations/004_health_checks.sql
create table health_checks (
  id          uuid primary key default gen_random_uuid(),
  checked_at  timestamptz not null default now(),
  service     text not null,
  status      text not null check (status in ('ok', 'degraded', 'fail')),
  latency_ms  integer,
  error_msg   text
);
alter table health_checks enable row level security;
create index health_checks_service_checked_at on health_checks (service, checked_at desc);
```

## Decisões de design importantes

- **Cookie admin_token** = valor de `ADMIN_SECRET` directamente (simples, aceitável para portfolio pessoal)
- **Vapi check** — usa `NEXT_PUBLIC_VAPI_PUBLIC_KEY` (não existe chave server-side). Só falha em 5xx. 401/403 são ok.
- **Cron path** em `vercel.json` usa `/ai-agent-voice/api/cron/health-check` — igual ao padrão do cron existente
- **Railway porta 8081** — separada da porta LiveKit. Verificar exposição pública no dashboard Railway.
- **Email FROM** `health@raphaelbruno.dev` — requer domínio verificado no Resend (já configurado via Cloudflare)
- **`checkSupabase`** queries tabela `health_checks` (correcta — criada pela migração 004)
