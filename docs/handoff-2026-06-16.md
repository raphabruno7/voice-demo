## Twilio — ✅ Concluído (esta sessão)

### Problema inicial
- Conta Twilio em Trial, 0 números comprados (user pensava ter comprado um)
- `twilio-agent/server.js` usava `@anthropic-ai/sdk` (sem `ANTHROPIC_API_KEY`)
- `app/api/twilio/twiml/route.ts` usava `ttsProvider="ElevenLabs" voice="bBNhdwrIjl4fcVYiRbT2"` → HTTP 402 no Free plan

### O que foi feito

**1. `twilio-agent/server.js` reescrito**
- Removido `@anthropic-ai/sdk`
- Gemini 2.0 Flash via `fetch` nativo SSE (sem SDK, Node 18+ built-in)
- Protocolo ConversationRelay mantido: `{type:"setup"}` → saudação; `{type:"prompt", voicePrompt}` → stream → `{type:"text", token, last}`
- Histórico de conversa em `{ role: "user"/"model", parts: [{text}] }` (formato Gemini)

**2. `app/api/twilio/twiml/route.ts` corrigido**
- `ttsProvider="ElevenLabs" voice="bBNhdwrIjl4fcVYiRbT2"` → `ttsProvider="amazon" voice="Polly.Ines-Neural" language="pt-PT"`
- Amazon Polly built-in no Twilio, sem BYOK, sem configuração extra

**3. Twilio API Key + TwiML App criados via REST**
- `TWILIO_ACCOUNT_SID`: já existia no Vercel (para WhatsApp sandbox) (já existia no Vercel para WhatsApp)
- `TWILIO_AUTH_TOKEN`: já existia no Vercel (para WhatsApp sandbox) (já existia no Vercel)
- API Key criada: `SK093a...` → `TWILIO_API_KEY` em Vercel
- API Key Secret → `TWILIO_API_SECRET` em Vercel
- TwiML App criada: `APe956...` → `TWILIO_TWIML_APP_SID` em Vercel
- Voice URL do TwiML App: `https://voice-demo-navy.vercel.app/api/twilio/twiml`

**4. Deploy do twilio-agent no Fly.io**
- Problema: ngrok sem authtoken; cloudflared (disponível) dá URL temporário — não serve para clientes acederem independentemente
- Decisão: Fly.io (free tier, CC só para verificação, não cobra para um processo Node leve)
- `flyctl` instalado via Homebrew
- `twilio-agent/Dockerfile` criado (node:20-alpine, `npm install --omit=dev`, PORT 8080)
- `twilio-agent/fly.toml` criado + ajustado: `auto_stop_machines='off'`, `min_machines_running=1`, `memory='256mb'`
- `flyctl secrets set GEMINI_API_KEY=...`
- `flyctl deploy` → `https://voice-demo-twilio-agent.fly.dev` (2 VMs, região `cdg`)
- `TWILIO_AGENT_WSS_URL=wss://voice-demo-twilio-agent.fly.dev` em Vercel

**5. Verificação end-to-end**
- `/api/twilio/token` → JWT 498 chars ✅
- `/api/twilio/twiml` → XML com `wss://voice-demo-twilio-agent.fly.dev` + Amazon Polly Inês ✅

## Fly.io — nota operacional

Após o primeiro `flyctl deploy`, as machines ficam `stopped` até receberem tráfego (auto_start=true). Para forçar arranque imediato: `flyctl machine start <id> --app voice-demo-twilio-agent`. Com `auto_stop=off` e `min_machines_running=1`, ficam a correr indefinidamente após o primeiro arranque.

**Re-deploy** (quando mudar `server.js` ou `system-prompt.txt`):
```bash
cd twilio-agent && flyctl deploy
```

| Constraint | Porquê |
|---|---|
| `NEXT_PUBLIC_VAPI_*` só inline via Git push para `main` | `vercel --prod` de branch `feat/*` quebra rotas raiz (basePath conflict) |
